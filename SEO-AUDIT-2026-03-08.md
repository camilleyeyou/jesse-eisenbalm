## Executive Summary

The site has made strong infrastructure progress since the original audit (46 → 58). Technical SEO is now the strongest category at 86/100. However, **content quality is the critical bottleneck** — scoring just 24/100, it single-handedly prevents the site from reaching a competitive score.

**What's working well:**
- Pre-rendered HTML for all pages (blog, about, FAQ, privacy)
- Valid structured data across all page types (Product, Organization, WebSite, Blog, BlogPosting, AboutPage, BreadcrumbList)
- Strong security headers including new CSP
- Clean URL structure with proper redirects (www→non-www, HTTP→HTTPS)
- 404 handling now returns proper HTTP 404 status codes
- All images have descriptive alt text, WebP format, and explicit dimensions
- EisenbalmShop.jsx is now lazy-loaded
- Scroll handlers use refs (not state) — no unnecessary re-renders

**Three problems dominate the remaining gap:**

1. **Content trust crisis (E-E-A-T: 24/100)** — No real human author, unverified charity claim, pseudo-scientific health claims without citations, no ingredient list, no return policy
2. **LCP blocked by video opacity gate (~3.5-5.0s)** — Hero poster/video hidden until JS fires `onLoadedData`
3. **Near-duplicate blog content (5+ clusters)** — 8+ URLs covering ~3 distinct topics, diluting ranking signals

### 5. Performance (Core Web Vitals) — 52/100 (up from 40)

#### Estimated Metrics

| Metric | Homepage | Blog Index | Blog Post | Threshold |
|---|---|---|---|---|
| LCP | POOR (3.5-5.0s) | NI (2.5-4.0s) | NI (2.5-3.5s) | ≤2.5s |
| INP | AT RISK (200-350ms) | GOOD (50-150ms) | GOOD (50-100ms) | ≤200ms |
| CLS | GOOD (0.05-0.10) | GOOD (0.05-0.10) | GOOD (0.05-0.10) | ≤0.1 |

#### LCP Bottlenecks

1. **Homepage hero video opacity gate** — `.hero-video { opacity: 0 }` until `onLoadedData` fires. Browser cannot count zero-opacity elements as LCP. Combined with `preload="metadata"` (delays video download), LCP is blocked until JS state update.
2. **No hero poster preload** — `/images/hero-poster.jpg` is the LCP fallback but not preloaded in `index.html`.
3. **Blog index scroll-reveal hides above-fold text** — Hero heading starts at `opacity: 0; transform: translateY(30px)` via JS reveal system.

#### INP Improvements (since last audit)

- Scroll handlers now use refs, not state (no re-renders on scroll) — GOOD
- `{ passive: true }` on all scroll listeners — GOOD
- BlogPage parallax uses direct DOM manipulation — GOOD

#### INP Risks

- Cart interactions re-render entire 1271-line EisenbalmShop component
- Three overlapping scroll listeners on homepage (parallax, reveal, resize)
- 196-line inline `<style>` block re-parsed on every render

#### CLS Improvements

- Blog card images: `width="640" height="360"` + `loading="lazy"` — GOOD
- Blog post covers: `width="1200" height="630"` — GOOD
- Product images: `width={1024} height={1536}` — GOOD
- Aspect-ratio containers used consistently — GOOD

#### Bundle & Code Splitting

| Check | Status |
|---|---|
| EisenbalmShop lazy-loaded | PASS (fixed since Round 3) |
| All routes use `React.lazy()` | PASS |
| Stripe SDK NOT in frontend bundle | PASS (server-side redirect flow) |
| `web-vitals` uses `onINP` (not deprecated FID) | PASS |

#### Priority Performance Fixes

| # | Fix | Expected Impact |
|---|---|---|
| P1 | Remove hero video opacity gate — show poster immediately | LCP 3.5-5.0s → 1.5-2.5s |
| P2 | Preload hero poster in `index.html` with `fetchpriority="high"` | LCP -0.5-1.0s |
| P3 | Change `preload="metadata"` to `preload="auto"` on hero video | Faster video paint |
