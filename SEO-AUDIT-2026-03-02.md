## Executive Summary

The site has a clear brand vision, solid infrastructure (Vercel, HTTPS, sitemap, robots.txt, llms.txt), and a strategic content pipeline. However, **three architectural problems are suppressing all organic traffic**:

1. A **Vercel CDN cache poisoning bug** is serving the HTML shell instead of the JS bundle at some edge nodes — the site is functionally broken for affected users
2. **41 blog post URLs** serve an empty 1,853-byte HTML shell with no unique content — Googlebot cannot index any blog posts
3. **Fabricated review data** (247 AI-generated reviews presented as real) in Product schema risks a Google manual action

The site currently shows 18 total impressions and 0 clicks in GSC. Fixing the rendering issues alone could unlock the entire blog content portfolio for indexation.

## P0 EMERGENCY — Fix Before Anything Else

### Vercel CDN Cache Poisoning (Site broken for some users)

The main JavaScript bundle `/static/js/main.1c09daba.js` is being served as `text/html` (1,853 bytes) instead of `application/javascript` (405 KB) at Vercel edge nodes.

```
content-type: text/html; charset=utf-8   ← WRONG
content-length: 1853                      ← WRONG (should be 405,570)
cache-control: s-maxage=31536000, immutable  ← Cached for up to 1 YEAR
x-vercel-cache: HIT
```

The `/(.*) → /200.html` rewrite in `vercel.json` caught the JS bundle URL on a first request, and Vercel's edge cached it with `immutable`. That edge node now returns broken HTML for every JS request. No React app loads, no images, no checkout — only the `<noscript>` fallback text.

**Fix — Step 1:** Update `vercel.json` to exclude static assets from the SPA rewrite:
```json
{
  "rewrites": [
    {
      "source": "/((?!static|images|videos|favicon|manifest|robots|sitemap|llms).*)",
      "destination": "/200.html"
    }
  ]
}
```

**Fix — Step 2:** Deploy immediately to bust the poisoned edge cache.

## CRITICAL Issues — Fix Immediately

### C1. Blog posts not pre-rendered — 41 pages unindexable

Every blog post URL serves the generic `200.html` shell — a 1,853-byte file with no unique title, no canonical, no description, and no schema. Googlebot sees the same homepage shell for all 41 posts.

GSC Coverage data: **12 pages remain "Discovered — currently not indexed"** as of Feb 23, 2026.

**Root cause:** `vercel.json` universal rewrite. The build generates pre-rendered HTML for `/`, `/about`, `/blog`, `/faq`, `/privacy-policy` — but **not** for individual blog post slugs.

**Fix:** Add a build script that fetches all posts from the API and writes `build/blog/[slug]/index.html` for each post with baked-in title, canonical, description, OG tags, and Article JSON-LD. Update `vercel.json` to serve these files before falling through to `200.html`.

---

### C2. Fabricated AggregateRating in Product schema — Manual action risk

`src/EisenbalmShop.jsx` lines 293–298 declares:
```json
"aggregateRating": { "ratingValue": "5", "reviewCount": "247" }
```
The testimonials section in the same file (line 893) explicitly labels them: *"Real generated reviews from personas."* Google's structured data policy prohibits review markup for AI-generated content.

**Fix:** Remove the `aggregateRating` block entirely from `src/EisenbalmShop.jsx` lines 293–298.

---

### C3. FAQPage schema — Policy violation

`src/FaqPage.jsx` lines 136–147 contains `FAQPage` structured data. Google restricted `FAQPage` rich results to **government and healthcare authority sites only** as of August 2023. This site does not qualify.

**Fix:** Remove the entire `FAQPage` JSON-LD block from `src/FaqPage.jsx`.

---

### C4. All 404s return HTTP 200 (soft 404s)

The universal `vercel.json` rewrite means every non-existent URL returns `200 OK`. The `build/404.html` file exists but is never served. No `<Route path="*">` in `src/App.jsx`.

**Fix:** Add a wildcard catch-all in `vercel.json` serving `build/404.html` with `"status": 404`. Add a 404 route in `src/App.jsx`.

---

### C5. Live API key committed to documentation

`BLOG_AUTOMATION_SYSTEM.md` line 386 contains a plaintext credential:
`adb66a122a92b46278069f15d1d5b376103d80ea54745e0a29317896261cced8`

**Fix:** Rotate this key immediately. Replace with `[REDACTED]` in the documentation.

### C6. Touch targets are 20x20px — far below 48x48px minimum

The shopping cart button and hamburger menu toggle both measure exactly **20x20px** — the bare icon with no padding. These are the two most important mobile controls on every page load.

**Fix:** Add `p-3` to both `<button>` elements in the nav in `src/EisenbalmShop.jsx`.

---

### C7. Heading hierarchy skips H1 → H3 for the product name

The product section heading (`JESSE A. EISENBALM`) is tagged `<h3>` (`src/EisenbalmShop.jsx` ~line 752) immediately after the `<h1>` hero. The "Crafted for the Human Experience" section also uses `<h3>` instead of `<h2>`.

**Fix:** Change both to `<h2>`.

## HIGH Issues — Fix Within 1 Week

### H1. Charity name never disclosed

Every page claims "100% proceeds to charity" but no charity is named — not in FAQ, About, homepage, or `llms.txt`. This makes the claim unverifiable.

**Fix:** Name the specific charity on every page that makes this claim.

---

### H2. About page title declares "AI Marketing Team"

`src/AboutPage.jsx` line 99: `"About Jesse A. Eisenbalm | Meet Our AI Marketing Team"`. This appears in Google SERPs and signals to both users and search engines that there are no real humans behind the brand.

**Fix:** Replace with a human-centered brand title.

---

### H3. Product images are 2–2.7 MB PNG files

| Image | Size |
|---|---|
| `eisenbalm-1.png` | **2.1 MB** |
| `eisenbalm-2.png` | **1.64 MB** |
| `eisenbalm-3.png` | **2.67 MB** |
| Grid images (8 total) | **1.7–2.8 MB each** |
| Total grid payload | **~22 MB** |

Served without WebP conversion, without lazy loading, without explicit `width`/`height` attributes.

**Fix:** Convert all images to WebP at 80% quality (expect 70–85% size reduction). Add `width`/`height` attributes. Add `loading="lazy"` to below-fold images and `fetchpriority="high"` to the primary product image.

---

### H4. JS bundle served without compression

```
content-length: 424762
# No content-encoding header (no gzip or brotli)
```

424KB of uncompressed JavaScript on first load is a significant LCP bottleneck.

**Fix:** Verify `compress` is enabled in Vercel project settings.

---

### H5. No human founder or named team member

The About page lists AI personas. The author for all blog posts is the brand name. Google's September 2025 QRG update requires real human identity signals for E-E-A-T.

**Fix:** Add a named founder, formulator, or team member with a brief bio.

### H6. No return/refund policy

The site processes real payments via Stripe but has no return policy page. This is a QRG trustworthiness gap and may violate Stripe's merchant terms of service.

**Fix:** Add `/return-policy` and link from footer and checkout.

### H9. `exploring-beeswax-properties-for-skin-benefits` missing from sitemap

This is the site's **highest-impression page** (9 impressions, ranking ~position 61) and is **not** in the sitemap.

**Fix:** Verify it's published in Supabase and add it to the sitemap generator.

---

### H10. Hero video missing `muted` attribute

`src/EisenbalmShop.jsx` line 672: the `<video>` uses `autoplay` without `muted`. Chrome and Safari on mobile require `muted` for autoplay. The hero will render as a blank/static poster on most mobile devices.

**Fix:** Add `muted` as a prop (use a ref due to React's muted attribute bug).

---

### H11. Duplicate schema: JSON-LD Product + Microdata conflict

The homepage has both a JSON-LD Product block in `<head>` and legacy Microdata attributes (`itemscope`, `itemprop`) in the `#product` section HTML. The Microdata price markup is also malformed.

**Fix:** Remove all Microdata `itemscope`/`itemprop` attributes from the HTML — keep only JSON-LD.

---

### H12. Organization/Article `logo` points to product photo

Both the `Organization` schema and `Article` publisher use `eisenbalm-1.png` (a product photo) as the logo. Google requires a dedicated logo image.

**Fix:** Replace with `logo192.png` as an `ImageObject` with explicit `width` and `height` in both `src/EisenbalmShop.jsx` and `src/BlogPost.jsx`.

| FCP | ~2.5–4s | Good: <1.8s | FAIL |

| INP | >500ms (mobile) | Good: <200ms | FAIL |
| CLS | ~0.15–0.35 | Good: <0.1 | FAIL |

### Key Performance Issues

**LCP bottlenecks:**
- Hero poster image on external jsDelivr CDN (cross-origin latency, not preloaded)
- Hero video with `preload="auto"` competes for bandwidth with all other resources
- 2.1 MB product PNG is the largest single resource on the page
- JS bundle (424 KB uncompressed) must parse before React renders

**INP risk:**
- `setScrollY` called on every scroll tick → full re-render of a 1,100-line component
- Two concurrent scroll event listeners
- Parallax transforms computed in JSX (React re-render path) instead of direct DOM mutation

**CLS sources:**
- All 16 `<img>` tags and the `<video>` tag lack `width`/`height` attributes
- Dynamic blog post section loads without height reservation
- `js-reveal` system hides elements on hydration, then reveals them (perceived shift)
