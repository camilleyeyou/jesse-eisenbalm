## CRITICAL Issues -- Fix Immediately

### C1. Homepage `<div id="root">` has no pre-rendered body content
**Severity: CRITICAL | Impact: Homepage LCP POOR (est. 4-7s mobile), content invisible to non-JS crawlers**

The `prerenderHomepage()` function in `scripts/prerender-blog.js` (line ~200) injects meta tags and structured data into `<head>` but does NOT inject any body content into `<div id="root">`. The div is empty. Additionally, the `isPageReady` state in `src/EisenbalmShop.jsx` shows a full-screen black overlay until the hero video's `onCanPlayThrough` fires.

**Combined effect:** On the most important page of the site, users on slow connections see a black screen, and crawlers that don't execute JS see nothing.

**Fix (2 parts):**
1. Add body content injection to `prerenderHomepage()`:
```javascript
const homeBody = `<main>
  <h1>Jesse A. Eisenbalm | Premium Beeswax Lip Balm</h1>
  <p>Premium beeswax lip balm. Petrolatum-free barrier restoration. Limited Edition Release 001.</p>
  <p>$8.99 | 100% proceeds to charity | Stop. Breathe. Balm.</p>
</main>`;
html = html.replace('<div id="root"></div>', `<div id="root">${homeBody}</div>`);
```
2. Remove the video-dependent loading gate. Show page content immediately, let video fade in as enhancement.

### C2. Fabricated testimonials still on homepage
**Severity: CRITICAL | Impact: Google manual action risk under Sept 2025 QRG**

The homepage still displays "Generated Voices of Humanity" -- "Real generated reviews from personas navigating the complexity of the modern world." Under Google's September 2025 Quality Rater Guidelines, fabricated reviews are classified as deceptive content regardless of transparency labeling.

**Fix:** Replace with real customer reviews or remove the testimonials section entirely.

---

### C3. Hero video loaded from third-party CDN with loading gate
**Severity: CRITICAL | Impact: LCP blocked until video downloads from external origin**

The hero video and poster image are fetched from `cdn.jsdelivr.net` (GitHub raw), adding DNS + TLS + download latency. The `isPageReady` overlay blocks ALL visible content until `onCanPlayThrough` fires.

**Fix:** Self-host video/poster in `/public/videos/`. Remove `isPageReady` gate -- show content immediately, let video enhance progressively.

---

### C4. Soft 404s -- Non-existent pages return HTTP 200
**Severity: CRITICAL | Impact: Crawl budget waste, index pollution**

The `vercel.json` rewrite catches all unmatched paths and serves `200.html` with HTTP 200. The React 404 UI renders client-side but the HTTP status code is always 200.

**Fix:** Add Vercel edge middleware or `routes` config to return proper 404 status for unknown paths.

## HIGH Issues -- Fix Within 1 Week

### H1. Blog posts are thin content (~800-900 words)
**Severity: HIGH | Impact: Helpful Content System devaluation**

Both analyzed posts are under 900 words against a 1,500-word minimum for blog posts targeting organic search. The blog automation system (`BLOG_AUTOMATION_SYSTEM.md`) targets only 900-1,200 words. All posts use identical templated structure (intro -> 4 H2 sections -> FAQ -> CTA), creating strong AI-content signals.

**Fix:** Update blog automation target to 1,500-2,000 words. Vary post structures (listicles, comparisons, how-tos, deep dives). Add original data, first-hand experience, or sourced statistics.

### H2. No real human author identified
**Severity: HIGH | Impact: E-E-A-T failure for health/skincare claims**

All content attributed to "Jesse A. Eisenbalm" (brand name, not a person). BlogPosting schema uses `@type: Person` but with a brand name. About page lists only AI personas. No founder, no formulator, no credentialed expert anywhere on the site.

**Fix:** Add a named founder or team member with bio and credentials. Create an author page. Update BlogPosting schema author to a real `Person` entity.

### H3. Charity name never disclosed
**Severity: HIGH | Impact: Core trust claim unverifiable**

"100% of proceeds go to charity" appears on every page but no charity recipient is named, no evidence is provided, and the FAQ calls it a "verified philanthropic model" without verification.

**Fix:** Name the specific charity. Link to their website. Provide evidence (receipt, badge, letter).

### H4. Homepage H1 is "ARE THESE MY REAL LIPS?"
**Severity: HIGH | Impact: Primary on-page SEO signal wasted**

Zero keyword relevance. The most important heading on the site communicates nothing about the product to search engines.

**Fix:** Replace with a keyword-relevant H1 like "Premium Beeswax Lip Balm" or "Jesse A. Eisenbalm | Artisan Beeswax Lip Balm." Move creative tagline to a subtitle.

### H5. Blog post cover images cause CLS
**Severity: HIGH | Impact: CLS 0.15-0.30 on blog pages**

Blog card images in `src/BlogPage.jsx` and cover images in `src/BlogPost.jsx` lack `width` and `height` attributes. Pre-rendered `<img>` tags in blog HTML also omit dimensions.

**Fix:** Add `width` and `height` to all blog `<img>` tags. In `scripts/prerender-blog.js` line ~114, add dimensions to pre-rendered cover images.

### H6. Blog post LCP has 4-step network waterfall
**Severity: HIGH | Impact: Blog post LCP est. 3-5s mobile**

Rendering waterfall: HTML -> JS bundle -> API fetch for post data -> image discovery. The cover image URL cannot be discovered until JavaScript fetches post data from the API.

**Fix:** In `scripts/prerender-blog.js`, add `<link rel="preload" as="image" href="${post.cover_image}" fetchpriority="high" />` to pre-rendered blog post HTML.

### H7. Near-duplicate blog posts (5+ pairs)
**Severity: HIGH | Impact: Duplicate content dilution**

| Pair | Risk |
|---|---|
| `beeswax-vs-petroleum-better-choice-for-lip-health` / `beeswax-vs-petroleum-jelly-the-best-lip-care-choice` | High |
| `the-art-of-slow-beauty-rituals-mindful-lip-care` / `the-art-of-slow-beauty-rituals-embrace-mindful-lip-care` | High |
| `analog-ritual-in-the-digital-age-lip-balm-application` / `analog-rituals-in-the-digital-age-finding-peace` / `analog-rituals-in-the-digital-age-a-path-to-mindfulness` | High |
| `discover-premium-lip-balm-for-sale` / `discover-the-best-lip-balm-for-mindful-living` / `discover-serenity-with-premium-lip-balm` | Medium |

**Fix:** Consolidate each group into one canonical post with 301 redirects from weaker URLs.

### H8. About page title declares "AI Marketing Team"
**Severity: HIGH | Impact: SERP signals no real humans behind brand**

`src/AboutPage.jsx`: `"About Jesse A. Eisenbalm | Meet Our AI Marketing Team"`. This appears in search results and directly undermines human trust signals.

**Fix:** Replace with a human-centered brand title.

### H9. No return/refund policy
**Severity: HIGH | Impact: QRG trustworthiness gap for e-commerce**

The site processes real payments via Stripe but has no return policy page.

**Fix:** Add `/return-policy` and link from footer and checkout.

### H10. EisenbalmShop.jsx not lazy-loaded
**Severity: HIGH | Impact: Bundle bloat for all routes**

The 1,100+ line homepage component is eagerly imported in `src/App.jsx` (line 3) while all other routes use `lazy()`. This inflates the initial bundle for every page visit.

**Fix:** `const EisenbalmShop = lazy(() => import('./EisenbalmShop'));`
