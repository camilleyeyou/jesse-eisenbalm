## CRITICAL Issues — Fix Immediately

### C1. Homepage has NO SEO metadata in static HTML
**Severity: CRITICAL | Impact: Primary product page invisible to non-JS crawlers**

The homepage serves `200.html` (1,853 bytes) with no canonical tag, no meta description, no Open Graph tags, and no structured data. Unlike blog posts which now have pre-rendered `<head>` tags, the homepage has zero SEO signals until JavaScript executes and React renders.

Googlebot does render JS, but with delays (1-7 days for new content). Non-JS crawlers (Bing, social media scrapers, many AI engines) see an empty page.

**Fix:** Extend the pre-render approach to inject homepage meta tags into `build/index.html`:
- `<link rel="canonical" href="https://jesseaeisenbalm.com/">`
- `<meta name="description" content="...">`
- Open Graph and Twitter Card tags
- Product schema (name, price: $8.99, availability: InStock)
- Organization schema

---

### C2. Soft 404s — Non-existent pages return HTTP 200
**Severity: CRITICAL | Impact: Crawl budget waste, index pollution**

The `vercel.json` rewrite catches all unmatched paths and serves `200.html` with HTTP 200. The React 404 UI renders client-side but the HTTP status code is always 200.

**Fix:** Add Vercel edge middleware to check if a route is valid and return proper 404 status for unknown paths. Or use Vercel's `routes` config with explicit route definitions.

---

### C3. ~24 MB of unoptimized PNG images
**Severity: CRITICAL | Impact: Page weight ~24.5 MB, devastating LCP and CLS**

| Asset | Count | Total Size | Format |
|---|---|---|---|
| Product images | 3 | 6.1 MB | PNG |
| Grid images | 8 | 17.7 MB | PNG |
| Hero video | 1 | 732 KB | MP4 (CDN) |
| **Total** | | **~24.5 MB** | |

No images use WebP/AVIF, `loading="lazy"`, `width`/`height` attributes, `srcset`, or `<picture>` elements. All 11 images load eagerly and simultaneously.

**Fix:** Convert all PNGs to WebP (expected ~95% size reduction to ~1.5 MB total). Add `loading="lazy"` to below-fold images. Add explicit `width` and `height` attributes to all `<img>` tags.

## HIGH Issues — Fix Within 1 Week

### H1. Blog post body content not pre-rendered
Only `<head>` metadata is injected by `prerender-blog.js`. The `<body>` still contains an empty `<div id="root">`. Crawlers that don't execute JS won't see article text.

**Fix:** Extend `prerender-blog.js` to inject article body HTML content.

### H2. Duplicate `<title>` tags in blog posts
Pre-rendered blog HTML has two `<title>` elements — the base CRA title plus the injected post-specific title.

**Fix:** Replace the existing `<title>` in the shell rather than appending a second one.

### H3. No code splitting — 416 KB JS bundle
All routes (About, FAQ, Blog, Admin, Privacy) are statically imported in `App.jsx`. Visitors to the homepage download everything.

**Fix:** Use `React.lazy()` for non-homepage routes. Expected ~30-50% bundle reduction.

### H4. Scroll handler triggers React re-render every frame
`setScrollY(window.scrollY)` on every scroll event causes full component re-renders. Major INP risk on mobile.

**Fix:** Use refs instead of state for scroll-driven transforms.

### H5. No `preconnect` or `preload` resource hints
Hero video poster loads from `cdn.jsdelivr.net` with no preconnect hint. LCP delayed 500ms-1s.

**Fix:** Add `<link rel="preconnect" href="https://cdn.jsdelivr.net" crossorigin>` to `index.html`.

### H6. Duplicate Product schema (JSON-LD + Microdata)
Homepage has Product schema via JSON-LD and also Microdata attributes (`itemScope`, `itemType`, `itemProp`) on HTML elements. Can confuse parsers.

**Fix:** Remove Microdata attributes, rely on JSON-LD only.

### H7. Charity name never disclosed
Every page claims "100% proceeds to charity" but no charity is named anywhere. The FAQ calls this a "verified philanthropic model" but provides no verification mechanism.

**Fix:** Name the specific charity. Link to their website. Provide evidence.

### H8. About page title declares "AI Marketing Team"
`src/AboutPage.jsx`: `"About Jesse A. Eisenbalm | Meet Our AI Marketing Team"`. This appears in SERPs and signals no real humans behind the brand.

**Fix:** Replace with a human-centered brand title.

### H9. Near-duplicate blog posts (3+ pairs)
| Pair | Risk |
|---|---|
| `beeswax-vs-petroleum-better-choice-for-lip-health` / `beeswax-vs-petroleum-jelly-the-best-lip-care-choice` | High |
| `the-art-of-slow-beauty-rituals-mindful-lip-care` / `the-art-of-slow-beauty-rituals-embrace-mindful-lip-care` | High |
| `analog-ritual-in-the-digital-age-*` / `analog-rituals-in-the-digital-age-*` | High |

**Fix:** Consolidate each pair into one canonical post with a 301 redirect from the weaker URL.

### H10. No return/refund policy
The site processes real payments via Stripe but has no return policy page. This is a QRG trustworthiness gap.

**Fix:** Add `/return-policy` and link from footer and checkout.

### H11. No human founder or named team member
The About page lists only AI personas. Google's September 2025 QRG update requires real human identity signals.

**Fix:** Add a named founder, formulator, or team member with a brief bio.

### H12. Organization/Article `logo` uses product photo
Both Organization schema and Article publisher use `eisenbalm-1.png` (a product photo) as the logo. Google recommends a dedicated logo image.

**Fix:** Replace with `logo192.png` as `ImageObject` with `width`/`height`.
